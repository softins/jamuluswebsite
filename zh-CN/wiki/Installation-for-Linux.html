使用 Jamulus | 快速入门

为 Linux 安装

请确保你阅读了入门页面。

升级?您可能需要先 备份您的配置。

安装服务器？ 请先阅读服务器管理员指南。

Debian 和 Ubuntu

使用官方存储库（推荐）

我们为基于 Debian 和 Ubuntu 的发行版提供了一个存储库，允许与其他系统更新一起自动更新。要使用存储库：

  1. 打开你的终端（CTRL+ALT+T 应该有效）
  2. 仅限 Ubuntu - 启用 Ubuntu“universe”存储库(您可以使用 基于 GUI 的方法 或 基于 CLI 的方法)。
  3. 下载存储库安装脚本:
    curl https://raw.githubusercontent.com/jamulussoftware/jamulus/main/linux/setup_repo.sh > setup_repo.sh
  4. 使脚本可执行:
    chmod +x setup_repo.sh
  5. 运行安装脚本并安装 Jamulus:
    sudo ./setup_repo.sh && sudo apt install jamulus

手动安装 .deb 文件

  1. 下载适合您的架构的 Jamulus: amd64, armf or arm64
  2. 仅限 Ubuntu - 启用 Ubuntu“universe”存储库(您可以使用 基于 GUI 的方法 或 基于 CLI 的方法)。
  3. 通过打开命令行窗口更新 apt(CTRL+ALT+T 应该可以)并输入:sudo apt-get update
  4. 转到下载安装程序的位置并双击它，或使用命令行: sudo apt install ./jamulus_3.11.0_ubuntu_amd64.deb (或用于 Raspberry Pi 等，如上所示)
  5. 安装后，您可以删除该文件并关闭命令行窗口。

升级: 如果您需要将 Jamulus 升级到较新的版本并手动安装 Jamulus，只需下载新的 .deb 文件并按照上述步骤重新安装。

其他发行版

对于其他发行版上的安装程序，请参阅其包管理器和 Repology。如果您的发行版中不包含最新版本的 Jamulus，您可以 从源代码编译 Jamulus。另请注意贡献的 安装脚本。

设置你的硬件

Jamulus 客户端需要连接到正在运行的 JACK 服务器才能启动。您需要找出哪个是适合您的发行版的低延迟声音服务器。- 如果您的发行版使用 JACK，请参阅如何 使用 QjackCtl 配置 JACK。- 如果您的发行版使用 PipeWire，请参阅如何 配置 PipeWire。

使用 QjackCtl 配置 JACK

要运行 JACK 服务器，推荐的方法是使用 QjackCtl 。

  1. 启动 QjackCtl。您将看到 Qt JACK Control 实用程序主页。
  2. 按以下步骤配置声音硬件:
  • 将音频 接口 设置为您想要的 (列表中可能有多个 - 选择正确的一个，因为如果不停止 Jamulus 和 JACK 就无法更改它)。
  • 将采样率设置为 48000。
  • 首先将 帧/周期”设置为 128 ，将 周期/缓冲区 设置为 2

重新启动 JACK 以使新设置生效，然后从下面的 启动 Jamulus 部分继续。

配置 PipeWire

PipeWire 提供了自己的 JACK 服务器。启动 Jamulus 客户端时，PipeWire 会自动运行 JACK 服务器。但是，需要调整一些参数。您需要将音频接口设置为“专业音频”配置文件，并预先定义 PipeWire 的“速率”和“量子”参数。您可以在 PipeWire wiki 中找到有关配置的更多信息。

设置系统： 1. 打开 pavucontrol（音频混合器），在配置选项卡中为声音硬件选择 “pro-audio “配置文件。

  1. 在终端中使用以下两条命令强制 PipeWire 的速率和量子： 
    pw-metadata -n settings 0 clock.force-rate 48000
pw-metadata -n settings 0 clock.force-quantum 128

    此后，从下面的 启动 Jamulus 部分继续。

启动 Jamulus

配置 JACK 或 PipeWire 后，启动 Jamulus。

如果您遇到任何问题，请首先检查故障排除页面。
