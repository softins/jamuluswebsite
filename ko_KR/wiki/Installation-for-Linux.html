데비안 및 우분투

공식 저장소 사용하기 (권장됨)

우리는 사용자의 다른 시스템 업데이트와 함께 자동 업데이트를 허용하는 데비안 및 우분투 기반 배포를 위한 저장소를 제공합니다. 저장소를 사용하려면:

  1. 사용자의 터미널을 엽니다 (CTRL+ALT+T가 작동해야 함)
    • 우분투 전용 * - 우분투 “universe” 저장소를 사용하도록 설정합니다([GUI 기반 접근 방식])(https://askubuntu.com/a/148645) 또는 CLI 기반 접근 방식) 을 사용할 수 있습니다).
  3. 저장소 설정 스크립트 다운로드:
    curl https://raw.githubusercontent.com/jamulussoftware/jamulus/main/linux/setup_repo.sh > setup_repo.sh
  4. 스크립트를 실행 가능하게 만들기:
    chmod +x setup_repo.sh
  5. 설정 스크립트를 실행하고 Jamulus 설치하기:
    sudo ./setup_repo.sh && sudo apt install jamulus

.deb 파일 수동 설치하기

  1. 사용자의 아키텍처에 맞는 Jamulus 다운로드: amd64, armf or arm64
    • 우분투 전용 * - 우분투 “universe” 저장소를 사용하도록 설정합니다([GUI 기반 접근 방식])(https://askubuntu.com/a/148645) 또는 CLI 기반 접근 방식) 을 사용할 수 있습니다).
  3. 콘솔 창을 열고(CTRL+ALT+T가 작동해야 함) sudo apt-get update를 입력하여 apt를 업데이트합니다
  4. 설치 프로그램을 다운로드한 위치로 이동하여 두 번 클릭하거나, 명령줄을 사용합니다. sudo apt install ./jamulus_3.10.0_ubuntu_amd64.deb(또는 위와 같이 Raspberry Pi 등의 경우)
  5. 설치한 후에는 파일을 삭제하고 모든 콘솔 창을 닫을 수 있습니다.

업그레이드: Jamulus를 최신 버전으로 업그레이드하고 Jamulus를 수동으로 설치해야 하는 경우, 새 .deb 파일을 다운로드하고 위와 같이 다시 설치하면 됩니다.

다른 배포판

다른 배포판의 설치 프로그램은 패키지 관리자 및 Repology를 참조하십시오. 최신 버전의 Jamulus가 배포판에 포함되어 있지 않은 경우 소스에서 Jamulus 컴파일을 할 수 있습니다. 제공된 설치 스크립트도 참고하십시오.

사용자의 하드웨어 설정

QjackCtl로 JACK 구성하기

Jamulus 클라이언트를 실행하려면 JACK가 필요하지만, 먼저 구성해야 합니다. 권장되는 방법은 QjackCtl을 사용하는 것입니다.

  1. QjackCtl을 실행합니다. Qt JACK 컨트롤 유틸리티 기본 페이지가 표시됩니다
  2. 오디오 하드웨어를 다음과 같이 구성합니다(JACK에 대한 정확한 설정은 오디오 하드웨어의 기능에 따라 다름):
  • 오디오 인터페이스를 원하는 것으로 설정합니다(목록에 여러 개 있을 수 있음)
  • 샘플 속도를 48000으로 설정합니다
  • 프레임/주기 128로 설정하고 주기/버퍼를 처음에는 2로 설정합니다

새 설정을 적용하려면 JACK을 다시 시작합니다

Jamulus 시작하기

JACK이 실행되고 구성된 상태에서 Jamulus를 시작합니다.

사운드 끊김 문제(특히 JACK/QjackCtl에 의해 보고된 XRUN 오류)가 발생하면 더 큰 값(예: 256프레임 또는 3주기)을 설정해 보십시오. 낮은 프레임(예: 64 프레임)은 더 나은 성능을 제공할 수 있지만, 더 많은 사운드 문제를 가져올 수 있습니다. 그렇지 않으면 문제 해결 페이지를 참조하십시오.
