Servrar - Registrerade

Varför visas inte min server i listan? Varför registreras den inte?

Om du är korrekt registrerad (du kan se det här) och du eller dina vänner inte kan se din server, kan du behöva vänta eller starta din klient med alternativet --showallservers och försöka ansluta därifrån (se kommandoradsalternativ för hur du startar din klient med en konfigurationsoption). I vissa nätverkskonfigurationer kan du också behöva konfigurera din router för att vidarebefordra portnumret som används av Jamulus Server.

Om du ser ett meddelande som säger att servern är full, måste du vänta tills en plats blir ledig.

Du kan verifiera att din server är listad i den relevanta genren genom att kolla här.

Servrar - Oregistrerade

Jag kör min klient på samma maskin/nätverk som min server men kan inte ansluta till den

Anslut din klient till localhost eller 127.0.0.1. Om du kör klienten på en annan maskin än servern men på samma nätverk, anslut då till serverns lokala nätverksadress. Anslut inte via serverns offentliga (WAN) adress.

Vilken adress ska jag ge till andra så att de kan ansluta till min server?

Detta bör vara din offentliga IP-adress (hitta den med Google). Anslut din egen Jamulus-klient till den lokala IP-adressen för din server (localhost eller 127.0.0.1 om den är på samma maskin som din klient). Observera att din offentliga IP-adress kan ändras - se noteringen om “dynamisk DNS” i denna guide.

Vilka portnummer kan jag använda?

Du kan ställa in din server att lyssna på ett anpassat portnummer med alternativet --port. Om du gör detta, behöver du informera folk om vilket portnummer de ska ansluta till. De måste lägga till portnumret till adressen till din server i formatet [serverAddress]:[portNumber].

Ingen kan ansluta till min server - men jag kan ansluta lokalt

Först och främst, se till att du har aktiverat portvidarebefordran. Om du fortfarande inte kan se din server utanför ditt lokala nätverk kan det finnas problem med din internetleverantör (ISP).

Vissa ISP:er använder tekniker som Address plus Port (A+P) eller Carrier-grade NAT (CGN) för att spara adressutrymme (vanligtvis för IPv4). Det gör det omöjligt att vara värd för tjänster som Jamulus hemma eftersom de inte är synliga utanför ditt lokala nätverk.

För att upptäcka om CGN är problemet, gå till din routers konfigurationsskärm (vanligtvis 192.168.X.X) och titta på WAN-statussidan. Om din listade IPv4-adress inte matchar med din offentliga IPv4-adress (kolla den här), tyder det på att CGN kan vara aktiverat. Du kan försöka använda en IPv6-anslutning genom att starta klienter och server med flaggan -6 om IPv6 är tillgängligt för servern och alla klienter.

För att åtgärda problemet med IPv4, kontakta din ISP:s tekniska supportteam och informera dem om att du vill vara värd för en offentlig server hemma och därför behöver en riktig WAN-IP-adress. Du kan också nämna att CGN kan vara problemet. Dessutom kan du be om en statisk IP-adress för att undvika att använda DDNS.
