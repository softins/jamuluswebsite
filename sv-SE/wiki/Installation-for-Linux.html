Installation för Linux

Se till att läsa sidan Kom igång.

Uppgraderar du? Du kanske vill säkerhetskopiera din konfiguration först.

Installerar du en server? Läs först guiden för serveradministratörer.

Debian och Ubuntu

Använda det officiella arkivet (rekommenderas)

Vi tillhandahåller ett arkiv för Debian- och Ubuntu-baserade distributioner som möjliggör automatiska uppdateringar tillsammans med dina övriga systemuppdateringar. För att använda arkivet:

  1. Öppna din terminal (CTRL+ALT+T bör fungera)
  2. Endast Ubuntu - Aktivera Ubuntu “universe”-arkivet (du kan använda den grafiska metoden eller kommandoradsmetoden).
  3. Ladda ner installationsskriptet för arkivet:
    curl https://raw.githubusercontent.com/jamulussoftware/jamulus/main/linux/setup_repo.sh > setup_repo.sh
  4. Gör skriptet körbart:
    chmod +x setup_repo.sh
  5. Kör installationsskriptet och installera Jamulus:
    sudo ./setup_repo.sh && sudo apt install jamulus

Installera .deb-filerna manuellt

  1. Ladda ner Jamulus för din arkitektur: amd64, armhf eller arm64
  2. Endast Ubuntu - Aktivera Ubuntu “universe”-arkivet (du kan använda den grafiska metoden eller kommandoradsmetoden).
  3. Uppdatera apt genom att öppna ett terminalfönster (CTRL+ALT+T bör fungera) och skriv: sudo apt-get update
  4. Gå till den plats där du har laddat ner installationsprogrammet och dubbelklicka på det, eller använd kommandoraden: sudo apt install ./jamulus_3.11.0_ubuntu_amd64.deb (eller för Raspberry Pi osv. enligt ovan)
  5. När installationen är klar kan du radera filen och stänga eventuella terminalfönster.

Uppgraderingar: Om du behöver uppgradera Jamulus till en nyare version och installera Jamulus manuellt, ladda ner den nya .deb-filen och installera om enligt ovan.

Andra distributioner

För installationer på andra distributioner, se deras pakethanterare och Repology. Om en uppdaterad version av Jamulus inte ingår i din distribution kan du kompilera Jamulus från källkoden. Notera också de bidragna installationsskripten.

Ställ in din hårdvara

Jamulus-klienten behöver ansluta till en körande JACK server för att starta. Du behöver ta reda på vilken ljudserver med låg latens som används i din distribution.. - Om din distribution använder JACK, se hur du konfigurerar JACK med QjackCtl.. - Om din distribution använder PipeWire, se hur du konfigurerar PipeWire.

Konfigurera JACK med QjackCtl

För att köra en JACK server är den rekommenderade metoden att använda QjackCtl.

  1. Starta QjackCtl. Du kommer att se Qt JACK Control-verktygets huvudsida.
  2. Konfigurera din ljudhårdvara enligt följande:
  • Ställ in ljudgränssnittet till det du vill använda (det kan finnas flera i listan - välj rätt, eftersom detta inte kan ändras utan att stoppa Jamulus och JACK).
  • Ställ in samplingsfrekvensen till 48000.
  • Ställ in Frames/Period till 128 och Periods/Buffer till 2 till att börja med.

Starta om JACK för att de nya inställningarna ska träda i kraft och fortsätt från avsnittet Starta Jamulus nedan.

Konfigurera PipeWire

PipeWire tillhandahåller sin egen JACK-server. När du startar Jamulus-klienten kör PipeWire automatiskt en JACK-server. Det finns dock några parametrar som behöver justeras. Du behöver ställa in ditt ljudgränssnitt till “pro-audio”-profilen och definiera PipeWires “rate” och “quantum”-parametrar i förväg. Du kan hitta mer information om konfigurationen i PipeWire-wikin.

För att ställa in ditt system: 1. Öppna pavucontrol (ljudmixern) och välj “pro-audio”-profilen för din ljudhårdvara i konfigurationsfliken. 2. Använd följande två kommandon i en terminal för att tvinga PipeWires hastighet och kvantum:

pw-metadata -n settings 0 clock.force-rate 48000
pw-metadata -n settings 0 clock.force-quantum 128

Efter detta, fortsätt från avsnittet Starta Jamulus nedan.

Starta Jamulus

Med JACK eller PipeWire konfigurerat, starta Jamulus.

Om du har några problem, kolla först felsökningssidan.
